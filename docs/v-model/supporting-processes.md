# Supporting Processes

## 형상관리

| 형상 항목 | 식별 기준 | 저장 위치 |
| --- | --- | --- |
| 요구사항과 추적성 | WC-STK/SYS/SWR ID | `docs/v-model/` |
| STM32 소스 | 파일 경로와 Git commit | `Core/` |
| MCU 설정 | CubeMX 버전과 파일 hash | `TryMotorJoystick.ioc` |
| 시험 명세와 결과 | WC-UT/IT/ST ID | `docs/v-model/`, 시험 첨부 자료 |
| 외부 모듈 인터페이스 | 신호명, 전압, 주기, 데이터 형식 | `architecture.md` |

기준선에는 요구사항, 소스, CubeMX 설정, 시험 결과와 도구 버전을 함께 포함한다. 본 프로젝트의 확인 가능한 도구 기준은 STM32CubeMX 6.11.0, STM32Cube FW_F1 1.8.5이다.

## 변경관리

변경 요청은 다음 필드를 가진다.

- Change ID, 요청자, 변경 사유와 우선순위
- 영향받는 요구사항, 인터페이스, 코드, 시험
- 위험 분석과 회귀 시험 범위
- 검토자, 승인 상태, 구현 commit
- 검증 결과와 종료 판정

## 문제관리

결함은 재현 조건, 기대 결과, 실제 결과, 심각도, 원인, 수정 내용과 회귀 시험을 기록한다. 안전 상태 진입 실패, 방향 반전, 장애물 신호 누락은 최상위 우선순위로 분류한다.

## 검토 체크리스트

- 요구사항이 단일 의미이며 시험 가능한가?
- 시스템 요구사항과 소프트웨어 요구사항이 양방향으로 연결되는가?
- 모든 안전 관련 분기에 정상·경계·고장 시험이 존재하는가?
- 변경된 핀맵과 CubeMX 설정이 소스와 일치하는가?
- 시험 결과가 사용한 commit과 도구 버전을 식별하는가?
